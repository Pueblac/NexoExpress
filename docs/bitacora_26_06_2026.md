# Bitácora de Sesión — NexoExpress

**Fecha:** 26-06-2026
**Duración:** ~3 horas
**Proyectos tocados:** NexoExpress, BitácoraExpress, ActaExpressWeb

## 📋 Resumen Ejecutivo
Se realizó una reestructuración arquitectónica masiva del ecosistema. Se formalizó la visión en 3 capas (Recolección, Análisis, Reportería), se definió la estrategia para el motor RAG utilizando Firestore Vector Search y se establecieron límites estrictos de privacidad. También se crearon flujos automatizados de documentación con generación de diagramas Mermaid a Word (.docx) y se sincronizaron los repositorios con GitHub.

## 🎯 Objetivos de la Sesión
- Definir la viabilidad técnica del sistema RAG ("NotebookLM" propio).
- Actualizar los esquemas de datos para reflejar proyectos, ciclos de vida y privacidad.
- Conectar los repos locales a GitHub mediante PAT.
- Implementar un flujo automatizado de diagramación a Word.

## 🏛️ Decisiones de Arquitectura / Ecosistema
- **Modelo de 3 Capas**: El ecosistema se divide oficialmente en Recolección (las 3 apps), Análisis (RAG) y Reportería.
- **RAG Vectorial**: Se descartó la lectura bruta de textos por costos. Se usará *Firestore Vector Search* con embeddings generados vía Cloud Functions.
- **Privacidad Local**: BitácoraExpress tendrá un filtro local (horarios laborales / pausa) para no enviar datos privados a Firestore. Firestore es un "Cerebro Aislado" (estrictamente personal por `ownerId`).
- **Ciclo de Vida**: Los proyectos ahora tienen `fechaInicio` y `fechaFin`. Al cerrarse, su síntesis se hereda para evitar amnesia sin sobrecargar los vectores históricos.
